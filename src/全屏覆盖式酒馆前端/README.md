# 全屏覆盖式酒馆前端（准备区）

本目录当前是“脚本挂载到顶层 body 的全屏 iframe”骨架，重点是保证后续改造不破坏既有“每轮后总结写回世界书”链路。

## 目录

- `宿主脚本/`
  - `index.ts`: 宿主入口，创建并挂载顶层全屏 iframe，并在其内挂载 Vue 覆盖层；提供右下角“打开/关闭覆盖页”按钮。
  - `事件同步.ts`: 监听并同步核心原生事件（`MESSAGE_SENT`、`MESSAGE_RECEIVED`、`GENERATION_ENDED`、`WORLDINFO_UPDATED`）。
    - 同时采集“发送前原始输入”，用于覆盖层展示。
  - `原生发送.ts`: 接收“发送请求”，将输入写入原生输入框后点击酒馆原生发送按钮，不走 `createChatMessages`/`generateRaw` 替代主发送。
- `覆盖界面/`
  - 覆盖层 Vue 组件（由宿主脚本挂载到顶层 iframe 内）。
- `共享/协议.ts`
  - 共享事件名、状态结构、发送桥接载荷类型。

## 当前保证

- 主发送链路仍由酒馆原生按钮触发（覆盖层只请求，不直发）。
- 自定义层仅作为展示/控制入口，不直接替代酒馆底层会话机制。
- 已预留并打通宿主事件到覆盖层的同步通道。
- 默认隐藏原生聊天 UI，但保留开场白（第 0 楼）可见。

## 下一步建议

1. 为覆盖层接入真实对话渲染（注意区分用户原始输入与正则改写后文本）。
2. 增加轮次状态机，显式区分 dry_run 与正式阶段，避免误判“每轮结束”。
3. 补充“是否显示原生聊天 UI”的设置项（默认隐藏），便于调试。
